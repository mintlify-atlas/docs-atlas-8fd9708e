## Complete Model Comparison

Omnilingual ASR offers 27 models across four families: W2V (self-supervised), CTC (parallel ASR), LLM (autoregressive ASR), and Zero-Shot models.

<Note>
All VRAM and speed metrics measured on **A100 GPU** with **BF16 precision**, batch size 1, and 30-second audio (unless noted otherwise).
</Note>

## Model Families Overview

<CardGroup cols={2}>
  <Card title="W2V Models" icon="wave-square">
    **Self-Supervised Learning (SSL)**
    
    Pre-trained audio encoders producing contextualized embeddings. Useful as starting points for custom architectures.
    
    - 4 sizes: 300M, 1B, 3B, 7B
    - No direct transcription
    - Foundation for CTC/LLM models
  </Card>
  
  <Card title="CTC Models" icon="forward-fast">
    **Parallel ASR**
    
    High-speed speech recognition with parallel generation. Ideal for production deployments requiring throughput.
    
    - 4 sizes × 2 versions = 8 models
    - 16x-96x faster than real-time
    - No language conditioning
  </Card>
  
  <Card title="LLM Models" icon="brain">
    **Autoregressive ASR**
    
    State-of-the-art accuracy with language conditioning. Standard and Unlimited length variants.
    
    - 4 sizes × 3 variants = 12 models
    - Optional language conditioning
    - Unlimited length support (v2)
  </Card>
  
  <Card title="Zero-Shot" icon="wand-magic-sparkles">
    **In-Context Learning**
    
    Transcribe unseen languages using 1-10 audio-text example pairs.
    
    - 1 model (7B)
    - Requires context examples
    - Ideal for low-resource languages
  </Card>
</CardGroup>

  - Unlimited: Long-form content (podcasts, lectures, meetings)

## Performance Metrics

### Speed Comparison (Real-Time Factor)

<Note>
**RTF (Real-Time Factor)**: Time to process 1 second of audio. Lower is faster.
- RTF = 0.001: 1000x faster than real-time (1s audio in 0.001s)
- RTF = 1.0: Real-time processing (1s audio in 1s)
- RTF = 0.092: ~11x faster than real-time (1s audio in 0.092s)
</Note>

| Model Family | RTF Range | Speed vs Real-Time | Relative to LLM 7B |
|--------------|-----------|-------------------|--------------------|
| **CTC 300M** | 0.001 | 1000x faster | 96x faster |
| **CTC 1B** | 0.002 | 500x faster | 48x faster |
| **CTC 3B** | 0.003 | 333x faster | 32x faster |
| **CTC 7B** | 0.006 | 167x faster | 16x faster |
| **LLM (all)** | 0.090-0.093 | ~11x faster | 1x (baseline) |
| **LLM Unlimited (30s)** | 0.092-0.097 | ~11x faster | ~1x |
| **LLM Unlimited (15min)** | 0.206-0.208 | ~5x faster | ~0.5x |
| **Zero-Shot** | 0.194 | ~5x faster | ~0.5x |

## Technical Details

### Architecture Components

<Tabs>
  <Tab title="Wav2Vec2 Encoder">
    **Feature Extractor**:
    - CNN-based architecture
    - Downsampling: ~320x (16kHz → 50Hz)
    - Output: Frame-level features
    
    **Transformer Encoder**:
    - Sizes: 12/24/36/48 layers (300M/1B/3B/7B)
    - Dimensions: 1024/1280/2048/2048
    - Self-attention with positional encoding
    - Output: Contextualized audio embeddings
  </Tab>
  
  <Tab title="CTC Projection">
    **Linear Projection**:
    - Input: Audio embeddings (1024/1280/2048-dim)
    - Output: Vocabulary logits (9,812 or 10,288-dim)
    - No hidden layers (direct projection)
    
    **CTC Decoding**:
    - Parallel prediction over all frames
    - Blank token for alignment
    - Duplicate removal
    - Greedy argmax selection
  </Tab>
  
  <Tab title="Llama Decoder">
    **Projection Layer**:
    - Maps audio embeddings to 4096-dim Llama space
    
    **Llama Transformer**:
    - Decoder-only architecture
    - 4096-dimensional hidden states
    - Autoregressive generation
    - Beam search decoding
    
    **Final Projection**:
    - 4096-dim → vocab size
    - Produces token probabilities
  </Tab>
  
  <Tab title="Zero-Shot Context">
    **Context Processing**:
    - 10 audio-text pairs (fixed)
    - Each audio → Wav2Vec2 → embeddings
    - Each text → text embeddings
    - Interleaved with special tokens
    
    **Decoder Input**:
    - Context embeddings (10 pairs)
    - Target audio embeddings
    - Autoregressive generation conditioned on context
  </Tab>
</Tabs>

### Input Preprocessing

All models use the same preprocessing pipeline:

1. **Audio Decoding**: WAV, FLAC, MP3, etc. → raw waveform
2. **Resampling**: Any sample rate → 16kHz
3. **Channel Mixing**: Stereo/multi-channel → mono
4. **Normalization**: Amplitude normalization
5. **Length Validation**: Check max length constraints

```python
# From pipeline.py preprocessing
- Decode audio (if file/bytes)
- Resample to 16kHz
- Convert to mono
- Normalize amplitude
- Validate max length (40s for CTC/LLM, 60s for ZS)
```

## Common Limitations

<AccordionGroup>
  <Accordion title="Audio Length Constraints">
    - **CTC/LLM Standard**: 40 seconds maximum
    - **LLM Unlimited**: No limit (processes in 15s segments)
    - **Zero-Shot**: 60 seconds max (30s recommended for context)
    
    **Workaround**: Split longer audio or use Unlimited models
  </Accordion>
  
  <Accordion title="No Punctuation or Capitalization">
    Models output spoken-form text without punctuation or capitalization.
    
    **Workaround**: Use third-party punctuation restoration libraries like [deepmultilingualpunctuation](https://github.com/oliverguhr/deepmultilingualpunctuation)
  </Accordion>
  
  <Accordion title="Limited Script Support">
    While supporting 1,600+ languages, some rare scripts may have limited training data.
    
    **Workaround**: Use zero-shot model with examples in target script
  </Accordion>
  
  <Accordion title="No Real-Time Streaming (Current)">
    Unlimited models support segmented processing but the inference pipeline doesn't expose streaming API.
    
    **Future**: Underlying checkpoints can be extended for streaming applications
  </Accordion>
</AccordionGroup>
